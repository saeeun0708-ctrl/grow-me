# 나를 키워줘 — PRD (제품 요구사항 정의서)

> 버전 1.0 · 최종 수정 2026-06-05
> 한 줄 정의: **지인들이 골라준 단어로 내 캐릭터를 키우고, 그 결과로 "남이 보는 나"를 확인·자랑하는 바이럴 모바일 웹**

---

## 1. 제품 개요

### 1.1 핵심 컨셉
- 사용자(주인)는 자신만의 **공유 링크**를 받는다.
- 지인이 링크에 들어와 **이름을 적고**, "이 사람 하면 떠오르는 단어/어울리는 단어" **제시어 7개를 선택**해 **먹이주기**를 완료한다.
- 먹이를 받을 때마다 캐릭터가 성장하고, **가장 많이 받은 제시어에 해당하는 캐릭터**로 변신한다.
- 3단계까지 자라면 **독립시키기**로 결과를 확정하고, **인스타 자랑용 이미지**로 저장한다.

### 1.2 한 문장 가치 제안
> "친구들 눈에 비친 진짜 내 모습을, 귀여운 캐릭터 하나로 보여줘."

### 1.3 성공의 정의 (북극성 지표)
- **K-팩터(바이럴 계수) ≥ 1**: 한 명이 평균 1명 이상의 신규 주인을 데려온다.
  - 결과 이미지 공유 → 본 사람이 "나도 해볼래" → 신규 가입의 루프.
- 보조 지표: 먹이주기 완료율, 결과 이미지 저장률, 상세 언락 결제 전환율.

## 2. 타겟 사용자

| 구분 | 설명 |

| 주 타겟 | 10대 후반~20대 (SNS 자기표현·테스트 콘텐츠에 익숙한 세대) |

| 사용 맥락 | 인스타 스토리, 카톡 단톡방, 친구들 사이 유행 |

## 3. 핵심 사용자 흐름 (User Flow)

### 3.1 주인(Owner) 흐름
1. 링크 접속 → **간편 로그인**(카카오/구글) → 내 캐릭터 생성 (1단계, 작은 알 형태)
2. **내 공유 링크 복사 → 지인에게 뿌리기** (카톡/인스타 스토리/DM)
3. 지인들이 먹이를 줄 때마다 캐릭터 성장 알림 / 화면 갱신
4. 3단계 도달 → 원할 때 **[독립시키기]**
5. 결과 화면(1~5위 제시어 + 받은 수) → **이미지 저장** → SNS 자랑
6. (선택) **상세 언락 결제** → 누가 어떤 제시어를 골랐는지 전부 확인

### 3.2 지인(Sender) 흐름 — *로그인 불필요, 마찰 최소화*
1. 주인의 링크 접속 → "○○님에게 먹이주기" 화면
2. **내 이름 입력** (예: "단짝 민지")
3. **제시어 칩 중 7개 선택** (7개 채워야 완료 버튼 활성화)
4. **[먹이주기]** 버튼 → 완료 애니메이션 → "○○님 캐릭터가 성장했어요!" + "나도 만들기" CTA

## 4. 성장 규칙 (Growth Logic)

### 4.1 단계 (레벨)
| 레벨 | 조건 | 모습 |
|---|---|---|
| Lv.0 | 먹이 0회 (초기) | 8종 공통 **알** (눈 감은 씨앗) |
| Lv.1 | 먹이 1회 | 8종 공통 **새싹 아기** (눈 뜬, 머리 위 새싹) |
| Lv.2 | 먹이 2회 | 1위 카테고리 특징이 **절반** 드러난 어린이 |
| Lv.3 | 먹이 3회 이상 | 1위 카테고리의 **완성형 캐릭터** (소품·표정 완성) |

- Lv.3 이후 먹이를 더 받아도 **외형은 더 성장하지 않음** (제시어 카운트는 계속 누적).
- **독립시키기**는 Lv.3부터 가능.
- **캐릭터 종류 결정**: 누적된 모든 제시어를 카테고리별로 합산 → **1위 카테고리**의 캐릭터로 확정.
- 동점 시: 먼저 1위에 도달한 카테고리 우선(타임스탬프 기준).

### 4.2 먹이주기 규칙
- 1회 먹이 = 지인 1명이 제시어 7개 제출.
- **중복 방지**: 같은 기기/세션에서 동일 주인에게 중복 제출 제한(쿠키·localStorage 기준, MVP 수준).
- 한 명이 7개를 고르면 그 7개 제시어 각각 +1 카운트.

### 5.2 캐릭터별 디자인 특징 정의서 (클로드 디자인 전달용)

각 캐릭터는 **1·2·3단계 3종**으로 제작. 1단계는 8종 모두 동일한 공통 알 형태.

- **공통 1단계 (알)**: 손바닥만 한 동그란 파스텔 방울. 점 두 개 눈, 아주 작은 입. 표정은 호기심.

| 캐릭터 | 한 줄 소개 | 색감 | 3단계 핵심 특징 | 액세서리/포인트 |
|---|---|---|---|---|
| 똑똑이 | 뇌가 섹시해 | 차분한 블루 | **동그란 안경**을 쓴 지적인 표정 | 안경, 작은 책 또는 연필 |
| 멋쟁이 | 눈을 뗄 수 없는 비주얼 | 핑크/로즈 | 반짝이는 눈, 자신감 있는 미소 | 볼터치, 반짝이 별 |
| 불꽃이 | 열정! 식지 않는 에너자이저 | 레드/오렌지 | 머리 위 **작은 불꽃**, 힘찬 표정 | 불꽃 머리, 주먹 쥔 포즈 |
| 러블리 | 사랑받는게 일인 걸 | 크림/연노랑 | 가장 작고 동글, 발그레한 볼 | 새싹/리본, 살짝 웃는 눈 |
| 인기쟁이 | 어디든 빠지면 섭하지 | 라임그린 | 익살스러운 윙크, 메롱하는 혀 | 삐죽 머리, 웃긴 표정 |
| 포근이 | 곁에 있으면 마음이 놓여 | 라벤더/베이지 | 부드러운 미소, 포근한 인상 | 담요/머플러, 하트 |
| 재주꾼 | 못하는 걸 찾는 게 더 빨라 | 민트/멀티 | 여러 도구를 든 만능 느낌 | 셰프모자 또는 마이크 등 소품 |
| 신비 | 알수록 더 궁금해 | 퍼플/다크 | 반쯤 감은 눈, 묘한 분위기 | 별·달, 망토 또는 후드 |

> **참고**: 단순함을 위해 각 캐릭터 2단계는 "3단계 특징이 절반만 드러난(예: 안경이 작거나 흐릿한)" 중간 형태로 통일.

## 6. 결과 화면 & 공유 (바이럴 핵심)

### 6.1 무료로 보이는 것
- 확정된 캐릭터 (종류 + 3단계 모습)
- **1위 ~ 5위 제시어**와 각 **받은 횟수**
- 총 받은 먹이 수
- 캐릭터 이름/타입 (예: "당신은 **똑똑이**!")

### 6.2 결과 이미지 저장
- **인스타 최적 사이즈로 내보내기**: 스토리용 1080×1920, 피드용 1080×1350 중 선택.
- 이미지에 포함: 캐릭터, 1~5위 제시어 랭킹, 받은 수, **앱 이름·로고 워터마크**(출처 노출 = 자연 광고).
- 캡처가 아니라 **앱이 직접 예쁜 카드 이미지를 생성**(html-to-image 등)해서 저장.

### 6.3 독립시키기
- 3단계부터 활성화. 누르면 **더 이상 먹이 못 받음**(링크 비활성), 결과가 확정 고정.
- 독립 후 결과 화면 = 자랑·공유의 최종 형태.

## 7. 수익화 (Monetization)

### 7.1 핵심 모델 — 상세 결과 언락 (결제)
| 구분 | 무료 | 결제(언락) |
|---|---|---|
| 1~5위 제시어 | ✅ | ✅ |
| 전체 제시어 분포(6위 이하 포함) | ❌ | ✅ |
| **누가** 어떤 제시어를 골랐는지 (이름별 상세) | ❌ | ✅ |

- 결제 트리거: 결과 화면에서 "**누가 나를 똑똑하다고 했을까? 👀**" → 블러 처리된 이름 리스트 → [전체 보기 결제].
- 가격(초안): **건당 ₩990** (캐릭터 1개 상세 언락). 부담 없는 가격으로 결제 전환율 극대화.
- 결제 수단: **토스페이먼츠 / 페이플**(국내 카드·간편결제).

### 7.2 보조 (로드맵)
- 추가 캐릭터 스킨·결과 카드 테마 판매.
- 보상형 광고(먹이 부스트) — 초기엔 미적용, 유저 증가 후 검토.

## 8. 데이터 모델 (Supabase / PostgreSQL)

```
users
  id (uuid, pk)
  auth_provider     -- 'kakao' | 'google'
  provider_user_id
  nickname          -- 표시 이름
  share_slug        -- 공유 링크용 고유 슬러그 (예: ab12cd)
  character_code    -- 확정된 캐릭터 코드 (null이면 미확정)
  stage             -- 1 | 2 | 3
  is_independent    -- 독립시키기 여부 (bool)
  is_unlocked       -- 상세 언락 결제 여부 (bool)
  created_at

feedbacks            -- 먹이 1회 = 1 row
  id (uuid, pk)
  owner_user_id (fk → users.id)
  sender_name        -- 지인이 입력한 이름
  selected_words     -- text[] (제시어 7개)
  created_at

-- 집계는 feedbacks에서 실시간 계산하거나 word_counts 뷰/캐시로 관리
```

- **캐릭터 결정 로직**(서버/쿼리): `feedbacks.selected_words`를 펼쳐 카테고리 매핑 → 카테고리별 합계 → 최댓값 카테고리 = `character_code`.
- **단계 결정**: `feedback 수` = 레벨. 0=Lv.0(알), 1=Lv.1(새싹), 2=Lv.2, 3+=Lv.3(완성). 즉 `min(먹이수, 3)`.

## 10. 화면 목록 (Screens)

1. **랜딩/로그인** — 서비스 소개 1줄 + 간편 로그인
2. **내 캐릭터(주인)** — 현재 단계 캐릭터, 받은 먹이 수, 공유 버튼, (3단계 시)독립 버튼
3. **공유 시트** — 링크 복사 / 카톡 / 스토리 공유
4. **먹이주기(지인)** — 이름 입력 + 제시어 칩 7개 선택 + 먹이주기 버튼
5. **먹이 완료** — 성장 애니메이션 + "나도 만들기" CTA
6. **결과 화면** — 1~5위 랭킹, 받은 수, 이미지 저장, 상세 언락(블러+결제)
7. **상세 언락(결제 후)** — 이름별 제시어 상세
8. **결과 이미지 미리보기/저장**

---

## 11. 비기능 요구사항
- **단순함**: 지인은 3탭 안에 먹이주기 완료. 설명 없이도 직관적.
- **모바일 우선**: 세로 화면, 엄지 조작 최적화.
- **속도**: 첫 화면 1.5초 이내. 칩 선택 즉시 반응.
- **재미 요소**: 성장/완료 시 마이크로 애니메이션, 효과음(선택).
- **공유성**: 모든 결과는 "캡처/저장하고 싶게" 디자인.

---

## 12. MVP 범위 / 비범위

**포함(MVP)**
- 간편 로그인, 캐릭터 생성·성장(3단계), 먹이주기, 캐릭터 8종 결정 로직
- 결과 화면 + 1~5위 + 이미지 저장 + 워터마크
- 상세 언락 결제(1종) + 독립시키기

**제외(추후)**
- 스킨·테마 판매, 광고, 알림 푸시, 랭킹/탐색 피드

## 13. 리스크 & 대응
| 리스크 | 대응 |
|---|---|
| 어뷰징(자기 캐릭터에 셀프 먹이) | 기기·세션 중복 제한, 비정상 패턴 탐지(추후) |
| "그냥 캡처하면 결제 안 함" | 무료는 집계만, 결제는 **이름별 매핑** = 캡처 불가능한 정보 |
| 바이럴 실패 | 결과 카드 디자인·워터마크·"나도 만들기" CTA에 집중 |
| API/서버 비용 | AI 미사용(매핑은 규칙 기반) → 원가 거의 서버비뿐 |
